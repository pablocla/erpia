# Entitlements comerciales — Automation Hub

NOP gobierna el acceso al Automation Hub con una cascada de **suscripción comercial → feature flag → deny**.

---

## Modelos Prisma

| Modelo | Tabla | Rol |
|--------|-------|-----|
| `ProductoComercial` | `productos_comerciales` | Catálogo SKU, precio ARS, límite eventos/mes |
| `SuscripcionModulo` | `suscripciones_modulo` | Contrato activo por `empresaId` + `sku` |
| `UsageEvent` | `usage_events` | Contador mensual de eventos emitidos |

SKU principal: `automation.n8n_hub` → feature técnica `automation_n8n`.

---

## Cascada de acceso

```
canUseSku(empresaId, sku)
  ├─ SuscripcionModulo activa y vigente?
  │    ├─ límite mensual excedido → deny (402 usage_limit_exceeded)
  │    └─ OK → allow (source: subscription)
  ├─ SKU sin feature mapeada → allow (unrestricted)
  └─ isFeatureActiva(empresaId, featureKey)?
       ├─ true → allow (source: feature)
       └─ false → deny (module_not_entitled)
```

Implementación: `lib/platform/entitlements.ts`

---

## Registro de uso

Cada `emitToN8n` exitoso invoca `trackAutomationUsage()` → incrementa `UsageEvent` del mes UTC (`YYYY-MM`).

---

## API administrativa

### GET /api/platform/suscripciones

Lista suscripciones del tenant autenticado (JWT) con uso del mes actual.

### POST /api/platform/suscripciones

Alta/actualización (roles admin). Body ejemplo:

```json
{
  "sku": "automation.n8n_hub",
  "activo": true,
  "limiteEventosMes": 50000,
  "vigenciaHasta": null
}
```

## Seed demo

`POST /api/auth/demo` crea suscripción demo con límite ampliado vía `ensureDemoAutomationSubscription()`.

Catálogo base: `seedCommercialCatalog()` en `lib/platform/commercial-service.ts`.
