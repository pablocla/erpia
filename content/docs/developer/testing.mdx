## Comandos para Gemini

### Verificación completa (G11)

```bash
npm run verify          # 302 tests + build
npm run test:e2e        # 3 E2E Playwright (levanta dev server)
npm run test:smoke      # readiness + gaps
npm run test:erp        # cobertura módulos
npm run test:auth
npm run test:precios
```

```powershell
# Windows — script todo-en-uno
powershell -ExecutionPolicy Bypass -File scripts/run-tests-gemini.ps1
```

### Health-check (G9)

```bash
npm run dev   # terminal 1
npx tsx scripts/health-check.ts   # terminal 2
```

### Requisitos

- Node.js 20+
- Unit tests: **no** requieren `DATABASE_URL` (Prisma mockeado en `__tests__/setup.ts`)
- E2E smoke: mocks en `e2e/helpers/auth.ts` — no requiere DB
- E2E integración real / demo login: sí requiere `DATABASE_URL` + seed
- Build: `.env` mínimo para `prisma generate`

## Qué debe hacer Gemini al corregir código

1. `npm run test:smoke` → módulos críticos
2. `npm test` → suite completa
3. `npm run build` → compila
4. Si tocó auth: `npm run test:auth`
5. Si tocó precios/ventas: `npm run test:precios`
6. Si agregó servicio: `__tests__/<modulo>/<servicio>.test.ts`
7. Si tocó UI flujos: `npm run test:e2e`
