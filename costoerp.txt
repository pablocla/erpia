🔴 BLOQUEANTES ABSOLUTOS — Sin esto no puede salir a producción

  [B1] Tests de integración insuficientes
       → Solo 18 test files para 122 rutas API
       → Mínimo necesario: test para cada módulo crítico (ventas, caja, AFIP, stock)
       → Esfuerzo: 3-4 semanas de 1 dev sénior dedicado
       → Sin tests → primer día en prod hay data corrupta

  [B2] Apertura y cierre de caja con cuadre real
       → Existe la UI pero falta cuadre por medio de pago
       → Falta diferencia entre base de apertura y contado real
       → Sin esto ningún comercio puede cerrar su jornada correctamente
       → Esfuerzo: 1 semana

  [B3] Cierre de caja y factura AFIP en POS sin flujo completo
       → El POS emite factura pero falta el flujo: seleccionar cliente →
         calcular impuestos automáticos → multi-pago → emitir → actualizar CC
       → Hoy cada paso está separado, no hay un flujo POS unificado
       → Esfuerzo: 2 semanas

  [B4] Motor de precios con escalones y listas
       → Las listas de precio existen en DB pero no se aplican automáticamente
         en ventas (precio final = precioVenta siempre)
       → Sin esto: todos los clientes pagan lo mismo
       → Esfuerzo: 1 semana

  [B5] Gestión de usuarios con alta desde UI
       → Existe la tabla Usuario pero la página /dashboard/usuarios
         es básica (sin form de alta completo, sin asignación de rol visual)
       → Esfuerzo: 3 días

  [B6] Seed de datos base completo para todos los rubros
       → El seed.ts existe pero está incompleto (faltan maestros base:
         provincias, localidades rosarinas, condiciones de pago estándar,
         formas de pago, categorías por rubro)
       → Sin seed → cada empresa arranca con pantallas vacías sin poder operar
       → Esfuerzo: 1 semana

  [B7] Variables de entorno documentadas + deploy config
       → Falta docker-compose.yml, railway.toml o vercel.json de producción
       → Falta checklist de variables de entorno (DATABASE_URL, JWT_SECRET,
         claves AFIP, etc.) con validación al arrancar
       → Esfuerzo: 2-3 días

🟠 ALTA PRIORIDAD — Bloquean rubros específicos

  [A1] Envases / Retornables (distribuidora bebidas)
       → Requiere schema migration: TipoEnvase, MovimientoEnvase, SaldoEnvaseCliente
       → Sin esto: distribuidoras de bebidas no pueden operar (los cajones son activos)
       → Esfuerzo: 1 semana

  [A2] WhatsApp Business API para notificaciones
       → Diferenciador clave vs competidores
       → Eventos: pedido recibido, despachado, entregado, deuda, turno mañana
       → Requiere cuenta Meta Business o Twilio (ver lib/email/email-service.ts como base)
       → Esfuerzo: 2 semanas (incluyendo aprobación de templates en Meta)

  [A3] Plan Sanitario Veterinaria + recordatorios
       → Requiere schema PlanSanitario + cron job
       → Sin esto la veterinaria no puede programar vacunas automáticamente
       → Esfuerzo: 1 semana

  [A4] Conciliación bancaria automática (import extracto)
       → Hoy existe la UI pero el matching es manual
       → Falta parser de extractos CSV Galicia / BBVA / Santander / Macro / Provincia
       → Esfuerzo: 2 semanas

  [A5] POD completo con foto (Supabase Storage)
       → Firma y geo están implementados
       → Falta upload de foto del remito firmado a S3/Supabase
       → Esfuerzo: 3-4 días

  [A6] Cobro recurrente automático (membresías / gimnasio)
       → Requiere Mercado Pago Subscripciones API
       → Sin esto gimnasios deben cobrar manualmente cada mes
       → Esfuerzo: 2 semanas

🟡 MEDIA PRIORIDAD — Importantes pero no bloquean el lanzamiento

  [M1] PWA manifest.json + service worker
       → Para que Portal B2B y App Vendedor sean instalables en pantalla de inicio
       → Esfuerzo: 2 días

  [M2] Picking auto-generado desde pedidos confirmados
       → Hoy las listas de picking se crean manualmente
       → Debería generarse automáticamente al confirmar pedido B2B o de ruta
       → Esfuerzo: 3 días

  [M3] Gastronomía: workflow completo Pedido → Comanda → KDS → Factura
       → KDS existe pero falta el ciclo completo
       → Esfuerzo: 2 semanas

  [M4] RRHH: liquidación de comisiones de vendedores
       → Los vendedores no pueden ver sus comisiones calculadas automáticamente
       → Esfuerzo: 1 semana

  [M5] Agenda: recordatorio automático de turnos (cron job)
       → Falta job diario que detecte turnos del día siguiente → notifica
       → Esfuerzo: 3 días + WhatsApp API

  [M6] Salud: facturación por prestación + obras sociales
       → Alta de Profesional con prestaciones, tarifas por obra social
       → Liquidación diferida a la obra social
       → Esfuerzo: 3 semanas

🟢 POST-MVP — Para v1.1 o v1.2

  [P1] Integración Mercado Libre (sincronización stock + pedidos)
  [P2] Importación masiva de productos desde Excel/CSV
  [P3] Ruteo OSRM (optimización de recorrido de reparto)
  [P4] Peluquería: comisión por servicio configurable por profesional
  [P5] Farmacia: PAMI / obras sociales + troqueado ANMAT
  [P6] Agro: ticket de balanza, contrato de granos, portal productor
  [P7] Dashboard BI avanzado con comparativo vs mismo período año anterior
  [P8] Exportación contable a Tango / Bejerman / Xubio
  [P9] 2FA (TOTP o SMS)
  [P10] Multi-idioma (para expansión CL / MX con TES ya implementado)

OPCIÓN A — MVP MÍNIMO (1 rubro, 1 cliente piloto)
─────────────────────────────────────────────────
Objetivo: sacar 1 distribuidora de bebidas rosarina en producción
Tiempo: 8–10 semanas con 2 devs sénior dedicados

  Semana 1–2:  Cierre de caja completo + cuadre medios pago
  Semana 1–2:  Motor de precios con listas y escalones
  Semana 3:    Flujo POS unificado (cliente → ticket → cobro → factura AFIP)
  Semana 3–4:  Picking auto-generado desde pedidos
  Semana 4:    Envases/retornables (schema + UI)
  Semana 5:    Tests críticos: ventas, caja, stock, AFIP (cobertura mínima 60%)
  Semana 6:    Seed completo para distribuidora + onboarding wizard funcional
  Semana 7:    Deploy en producción (Railway o Render + Supabase)
  Semana 7:    AFIP en modo producción (certificados reales, punto de venta real)
  Semana 8:    Prueba piloto con 1 empresa real, soporte diario
  Semana 9–10: Corrección de bugs producción, performance bajo carga real

  Costo estimado (2 devs sénior × 10 semanas):
    → $16.000–20.000 USD si son empleados propios
    → $30.000–40.000 USD si se contratan en el mercado

7. RIESGOS REALES ANTES DEL LANZAMIENTO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔴 RIESGO ALTO
  - AFIP cambia los web services sin aviso (pasó en 2024 con ARCA)
    → Mitigación: mantener lib/afip actualizada, suscribirse al boletín AFIP
  - Bugs en facturación → multa al cliente
    → Mitigación: tests de integración reales contra homologación AFIP
  - Pérdida de datos por no tener backup automático
    → Mitigación: Supabase PITR ($25/mes) o pg_dump diario a S3
  - N+1 queries en listas con muchos registros → timeout
    → Mitigación: pagination server-side + índices en campos filtrados

🟠 RIESGO MEDIO
  - El JWT no expira correctamente en mobile → sesión infinita
    → Revisión del middleware.ts y refresh token
  - Rate limiting insuficiente → DDoS o brute force
    → lib/auth/rate-limiter.ts existe pero revisar cobertura
  - Transacciones DB sin rollback en operaciones críticas
    → Agregar prisma.$transaction() en: venta + movimiento stock + factura

🟡 RIESGO BAJO
  - Tailwind v4 con cambios de API → actualización de estilos
  - React 19 con features experimentales → algunos componentes pueden romper
