Lo que falta para “experiencia sana”:

Pre-vuelo al abrir POS
Si caja cerrada o AFIP mal → pantalla o banner bloqueante con un solo CTA (no carrito usable a medias).

Panel de suspendidas persistente
Lista con hora, ítems, total; click = recuperar (hoy solo modal).

Pendientes fiscales del turno
CAE pendiente / rechazado con link a comprobante.

Resumen de cierre visible
“Al cerrar turno: X ventas, Y en efectivo, Z pendiente de sincronizar”.

modoVendedor + sidebar reducido ya existen; falta que cajero no caiga en dashboard ejecutivo al loguearse.

PreviewFactura, DraftAlert, CajasAbiertasAlert — creados, no cableados en app/; el cajero no los ve.

4. Unificar “Mis tareas” vs “Pendientes del sistema”
Hoy mezcla conceptos:

Mis tareas → lista manual del usuario (bien para recordatorios).
Pendientes reales → repartidos entre dashboard, topbar, POS, localStorage.
Propuesta visual (una pantalla o dos pestañas):

[ Pendientes del sistema (12) ] [ Mis tareas (3) ]
Sistema: generado por API (/api/pendientes?rol=cajero agregando caja, AFIP, aprobaciones, stock…).
Mis tareas: lo actual.
En topbar: un solo icono con total (sistema + manual), no tres (tareas, tickets fake, campana).

7. Ejemplo concreto: día del cajero (experiencia sana)
Login → Home cajero (no dashboard ejecutivo).
Ve: “Caja cerrada” + “2 ventas suspendidas de ayer” + “AFIP OK”.
Abre caja → pendiente desaparece.
Entra POS → panel derecho con suspendidas; semáforos en verde.
Durante el día: CAE fallido aparece en pendientes con “Reintentar”.
Antes de irse: banner “Cierre Z pendiente” con checklist.
Topbar: un número total de pendientes, no tres iconos sueltos.
Eso no requiere features nuevas de negocio; requiere agregar API de pendientes por rol + vistas por rol + integrar componentes ya escritos.
